IDEA / Android Studio 中接入 Claude Code：替代 ProxyAI 的终端方案

更新于2026-07-01|AI工具
|浏览量:

ProxyAI 插件已无法访问，之前的 ProxyAI + DeepSeek V4 配置方案已失效。现在改为直接在 IDEA 内置终端中使用 Claude Code CLI，零插件、零配置，一条命令即可进入 AI 编程模式。本文记录这个方案的切换过程和实际使用方式。

背景：为什么放弃 ProxyAI

ProxyAI 插件之前通过 Custom OpenAI Provider 的方式在 IDEA 中接入 DeepSeek V4，实现了对话补全和代码补全。但 ProxyAI 服务已无法访问，该方案不再可用。

替代方案对比：

维度 ProxyAI（已失效） Claude Code CLI（当前方案）
模式 IDEA 插件 + API 配置 终端 CLI 直接使用
安装 插件市场安装 + 图形配置 npm install -g @anthropic-ai/claude-code
模型 自定义 API（DeepSeek 等） Anthropic Claude（默认）
上下文 受限于插件配置 原生 200K Token
适用场景 内嵌对话面板 + 代码补全 终端内编码、重构、调试、文件操作
IDE 依赖 强依赖 IDEA/Android Studio 独立 CLI，终端即可

第三步：常用操作速查

场景 命令
启动交互会话 claude
单次提问 claude -p "问题内容"
读取文件作为上下文 claude -p "分析 src/main.js 的架构"

🏷️ 常用技巧：

  • 在交互模式下可以直接引用文件路径让 Claude Code 读取
  • 用 /help 查看所有内置命令
  • 代码修改默认会直接写入文件，修改前 Claude Code 会展示 diff 并确认

第四步：终端快捷键与效率提升

在 IDEA 中可以配置终端快捷键，让 Claude Code 调用更顺手：

推荐的终端快捷键映射

操作 推荐快捷键 配置路径
打开/关闭终端 Ctrl+ö Keymap → Main Menu → Tools → Terminal
打开新建终端 Ctrl+Shift+ö Keymap → Terminal → Open in New Tab
终端全屏 Ctrl+Shift+F12 默认（隐藏所有面板，聚焦终端）

💡 我的个人配置：Ctrl+ö 快速呼出终端，进入后直接 claude 启动会话，用完后 Ctrl+F4 关闭标签页，回到编辑器。整个过程不到 2 秒。

终端复用建议

  • 每个项目一个终端标签页，用 IDEA 的终端 Tab 管理
  • Claude Code 会话适合长时间保持，用 /clear 清理上下文
  • 不需要时直接 Ctrl+C 退出，不消耗 API 额度

文件修改 ❌ 需手动复制 ✅ CLI 直接修改项目文件

跨 IDE 使用 ❌ 仅 IDEA 系 ✅ 任何编辑器/VSCode/终端

常见问题

❌ Claude Code 启动报错

原因：Node.js 版本过低或 npm 安装权限问题。

解决：

❌ 如何切换模型

Claude Code 默认使用 Anthropic Claude 模型。如果需要使用其他模型（如 DeepSeek），可以通过 Hermes Agent 作为代理层：

关于 Hermes Agent 的详细用法，参考同系列文章：Hermes Agent 日常使用与集成。

❌ 如何保留 ProxyAI 时期的 DeepSeek API 信息

DeepSeek V4 的 API Key 仍然有效（sk- 开头），如果后续需要继续使用 DeepSeek，可以通过以下方式：

  1. Hermes Agent 中配置 DeepSeek 作为编码提供商
  2. 直接调用 API：curl https://api.deepseek.com/chat/completions
  3. OpenAI 兼容库：配置 base_url=https://api.deepseek.com/v1 即可使用

总结

ProxyAI 插件的失效是一次被动迁移，但切换到 Claude Code 终端方案后，使用体验反而更加清爽——不需要额外插件、不需要图形配置、不需要在 IDEA 设置里反复调试参数。一条 claude 命令就能直接开始工作，尤其在重构代码、批量修改文件、管理 Git 流程时，CLI 方案比内嵌对话面板更贴合开发流程。

如果你之前在使用 ProxyAI 配置方案，直接卸载插件，打开终端输入 claude 即可开始新的工作流程。
